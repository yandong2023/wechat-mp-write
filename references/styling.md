# 公众号排版样式指南

## 设计原则

### 1. 可读性优先
- 字号适中，阅读舒适
- 行距宽松，不拥挤
- 段落分明，易扫描

### 2. 视觉层次
- 标题层级清晰
- 重点内容突出
- 留白得当

### 3. 品牌一致性
- 使用品牌主色
- 统一的视觉元素
- 固定的排版模式

## 字体规范

### 正文字体
```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', sans-serif;
```

### 字号设置

| 元素 | 字号 | 说明 |
|------|------|------|
| H1 标题 | 22px | 文章主标题，居中 |
| H2 副标题 | 18px | 章节标题，左对齐带边框 |
| H3 小标题 | 16px | 小节标题 |
| 正文 | 16px | 标准段落 |
| 引用 | 15px | 引用块略小 |
| 注释 | 14px | 表格、辅助信息 |

### 行高
- 正文：1.8 倍
- 标题：1.4 倍
- 引用：1.6 倍

## 颜色规范

### 主色调
- **品牌绿**：#1aad19（微信绿，用于强调、链接、边框）
- **正文黑**：#333333
- **标题黑**：#000000
- **辅助灰**：#666666
- **边框灰**：#dddddd
- **背景灰**：#f5f5f5

### 使用场景

| 颜色 | 场景 |
|------|------|
| #1aad19 | 链接、强调文字、H2 左边框、按钮 |
| #333 | 正文内容 |
| #000 | 标题、粗体 |
| #666 | 引用、注释、辅助信息 |
| #f5f5f5 | 引用块背景、代码块背景 |

## 间距规范

### 段落间距
- 段前：10px
- 段后：10px
- 段落间空行：一行（约 20px）

### 标题间距
- H1：上下 25px
- H2：上下 20px
- H3：上下 15px

### 列表间距
- 列表项之间：8px
- 列表整体上下：10px
- 列表缩进：20px

## 组件样式

### 引用块
```html
<blockquote style="border-left:3px solid #1aad19;padding:10px 15px;margin:15px 0;background:#f8f8f8;color:#666;font-size:15px;">
  引用内容
</blockquote>
```

**变体：**
- 💡 提示框：浅绿色背景
- ⚠️ 警告框：浅黄色背景  
- 📌 重点框：浅蓝色背景

### 表格
```html
<table style="width:100%;border-collapse:collapse;margin:15px 0;font-size:14px;">
  <thead>
    <tr><th style="background:#f5f5f5;padding:10px;border:1px solid #ddd;">表头</th></tr>
  </thead>
  <tbody>
    <tr><td style="padding:10px;border:1px solid #ddd;">内容</td></tr>
  </tbody>
</table>
```

### 分割线
```html
<hr style="border:none;border-top:1px solid #eee;margin:20px 0;" /\u003e
```

### 链接
```html
<a href="url" style="color:#1aad19;text-decoration:none;">链接文字</a>
```

### 图片
```html
<img src="url" style="max-width:100%;margin:15px 0;display:block;" /\u003e
```

**图片规范：**
- 宽度自适应（max-width: 100%）
- 上下间距 15px
- 居中对齐
- 添加阴影可选

## 特殊排版

### 文章开头

**场景引入：**
- 使用斜体或引用块
- 浅色背景
- 营造氛围

**数据展示：**
- 使用表格或大号数字
- 突出关键数据
- 加粗重点

### 文章主体

**步骤说明：**
```
第1步：标题
说明文字...

第2步：标题
说明文字...
```

**对比展示：**
- 使用两列表格
- 或左右并排布局

**列表呈现：**
- 无序列表：并列要点
- 有序列表：步骤流程
- 定义列表：术语解释

### 文章结尾

**总结框：**
- 边框包围
- 浅色背景
- 要点提炼

**行动号召：**
- 使用按钮样式或链接
- 突出显示
- 明确指引

**互动区域：**
- 投票、问卷
- 留言引导
- 转发提示

## 移动端适配

### 响应式设计
- 宽度自适应
- 图片不溢出
- 字体不过小

### 触摸优化
- 链接点击区域 > 44px
- 按钮足够大
- 间距适当

## 排版检查清单

发布前检查：

- [ ] 标题层级清晰（H1→H2→H3）
- [ ] 段落长度适中（不超过 5 行）
- [ ] 图片大小合适（< 2MB）
- [ ] 链接可点击且有效
- [ ] 表格在手机端可横向滚动
- [ ] 没有特殊字符乱码
- [ ] 代码块换行正确
- [ ] 引用块样式统一

## 常见问题

### Q: 为什么文字看起来拥挤？
A: 增加行高到 1.8，段落间距设为 10px

### Q: 图片显示不完整？
A: 添加 `max-width:100%` 和 `display:block`

### Q: 表格在手机上被压缩？
A: 使用 `min-width` 或添加横向滚动容器

### Q: 颜色在微信内置浏览器显示异常？
A: 使用微信官方推荐的颜色值，避免透明色

### Q: 字体在不同手机上不一致？
A: 使用系统字体栈，不强制特定字体
